## 5. 図の優先順位

### 優先度の判断基準

1. **高**: スライドの核心をなす図。テキストの Figure 番号に直接対応。ないと説明が困難。
2. **中**: 理解を助ける補足図。あると説明がスムーズになる。
3. **低**: 補強資料。時間があれば作成。

## 7. Mermaid で作る方針と SVG 化に mmdc を使う運用

### 基本フロー

```
1. .mmd ファイルを Mermaid 記法で作成
2. mmdc コマンドで .svg に変換
   $ mmdc -i <filename>.mmd -o <filename>.svg
3. 生成された .svg をスライドに組み込み
```

### Mermaid 図の種別マッピング

| 図の種類 | Mermaid 記法 | 対象ID |
|----------|-------------|--------|
| 概念・関係図 | `flowchart TD/LR` | S23-01, C3-03 |
| 再帰ツリー（コスト付き） | `flowchart TD` + ノード注記 | S23-03 |
| 手順ステップ | `flowchart TD` + サブグラフ | S23-02 |
| 関数グラフ（簡易版） | `xychart-beta` | C3-01, C3-02, C3-04 |

### ファイル命名規則

- `{内容を表すケバブケース名}.mmd` / `.svg`
- 例: `divide-and-conquer-paradigm.mmd` → `divide-and-conquer-paradigm.svg`

### 格納先

- `slides/images/` 配下に `.mmd` と `.svg` をペアで配置
- 既存の 7 図と同じディレクトリに追加

### Mermaid 以外のツールで作成する場合（Phase 3）

- ソースファイル（.py 等）も `slides/images/` に配置
- 最終成果物は必ず `.svg` で出力
- 再生成手順をソースファイル内のコメントに記載

### 10-2. 用語・記法の統一チェックリスト

| 項目 | 正しい扱い | よくある誤り |
|------|-----------|-------------|
| Θ vs O | Θ は「同じオーダー」、O は「上界」 | O を「最悪計算量」と同義で使う |
| O の等号 | `f = O(g)` は `f ∈ O(g)` の慣習的表記 | `O(g) = f` と逆に書く |
| 定数 c | c は n に依存しない定数 | c(n) のように n に依存させる |
| n₀ | n₀ は境界。n₀ 未満は何が起きてもよい | n₀ 未満も保証されると思い込む |
| log の底 | Θ 記法では底を無視（Θ(log₂ n) = Θ(ln n)） | 底の違いを Θ の区別に使う |
| マージソートの計算量 | 「常に Θ(n log n)」（best/worst/average 同じ） | 「最悪 O(n log n)」と만言う |

## 13. 作業上の注意（今回の範囲専用）

### 13-1. §2.3 と第3章の接続

§2.3（マージソートの計算量解析）で「T(n) = 2T(n/2) + Θ(n) = Θ(n log n)」と結論づけた上で、第3章でその Θ の定義を厳密に説明する、という流れになっている。

**注意点**:

- §2.3 の再帰木解析スライド（S23-03）では Θ を「同じオーダー」という直感的な意味で使用する。ここで Θ の厳密定義に立ち入らない
- 第3章の導入（G-1）で「§2.3 で Θ を使ったが、これは何を意味するのか？」と問いを立てることで、2つの章の接続を明示する
- `growth-of-functions.md` の「次回への接続」セクション（L647-649）が第4章への橋渡しであることを踏まえ、第3章内では §2.3 からの逆接続に集中する

### 13-2. O と Θ の使い分け

このスライドセットでは以下の方針を厳守する:

| 場面 | 使用する記法 | 理由 |
|------|-------------|------|
| マージソートの時間計算量 | **Θ(n log n)** | best/worst/average すべて同じだから。O(n log n) でも正しいが情報量が落ちる |
| 挿入ソートの最悪計算量 | **Θ(n²)** | 最悪ケースの正確な評価として |
| 挿入ソートの最善計算量 | **Θ(n)** | 最善ケースの正確な評価として |
| 挿入ソートの全体の上界 | **O(n²)** | 上界としての主張 |
| マージソートの空間計算量 | **Θ(n)** | 追加配列のサイズが n と同オーダーだから |
| 漸近的記法の説明での例 | **O / Θ / Ω を適材適所** | 定義の図示では明確に使い分ける |

**スライド内での表記**:

- 「計算量は O(n log n)」とは書かず、「計算量は **Θ(n log n)**」と書く
- 「最悪ケース O(n²)」ではなく、「最悪ケース **Θ(n²)**」と書く
- O は「上界であることを強調したい場合」にのみ使う

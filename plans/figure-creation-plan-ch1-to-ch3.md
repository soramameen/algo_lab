## 6. Mermaid で作るもの / 別手法が向くものの判断

### Mermaid（mmdc → SVG）を基本とする図

- **flowchart**: アルゴリズムの概念図、手順、関係性
  - C1-01, C1-02, C2-02, C2-05, C2-06, C3-03, C3-06
- **mindmap**: 応用領域の整理
  - C1-05
- **xychart**（簡易グラフ）: 関数の傾向比較（Phase 1 での暫定版）
  - C1-03, C3-01, C3-02

### 別手法が向く図（Phase 3 で対応）

- **精密な関数グラフ**: 座標軸のラベル、目盛り、交点の正確な表示が必要なもの
  - C3-01（Θ/O/Ω の境界線グラフ）
  - C3-02（増加率の対数スケール比較）
  - C1-03（クロスオーバーポイントの正確な表示）
  - **候補**: Python + matplotlib → SVG エクスポート
- **配列のステップ可視化**: Figure 2.3 のような詳細な配列インデックス付き図
  - C2-04
  - **候補**: HTML/CSS → スクリーンショット、または Python による描画

### 判断基準

| 観点 | Mermaid 向き | 別手法向き |
|------|-------------|-----------|
| 構造表現 | ○ ノード・エッジの関係 | — |
| 手順・フロー | ○ 段階的遷移 | — |
| 数値の正確なグラフ | △ 簡易版のみ可 | ○ 座標・目盛り精密制御 |
| 配列要素の位置合わせ | △ 簡易表現のみ | ○ ピクセル単位の制御 |
| SVG品質 | ○ mmdc で一貫出力 | ○ ツール依存（要確認） |

## 7. SVG化の運用方針

### 基本フロー

```
1. .mmd ファイルを Mermaid 記法で作成
2. mmdc コマンドで .svg に変換
   $ mmdc -i <filename>.mmd -o <filename>.svg
3. 生成された .svg をスライドに組み込み
```

### ファイル命名規則

- `{内容を表すケバブケース名}.mmd` / `.svg`
- 例: `insertion-sort-steps.mmd` → `insertion-sort-steps.svg`

### 格納先

- `slides/images/` 配下に `.mmd` と `.svg` をペアで配置

### Mermaid 以外のツールで作成する場合

- ソースファイル（.py 等）も `slides/images/` に配置
- 最終成果物は必ず `.svg` で出力
- 再生成手順をソースファイル内のコメントに記載

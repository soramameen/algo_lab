## 2. 共通デザイン原則

### 2-1. 基本: 図 7 : 文章 3

- **図のみのスライド**: 会話のアクセント。「ここは図で語る」場所
- **図＋短文（1行）**: 図の意味を1文で補足。箇条書き3行以内
- **文章中心**: 定義・証明・擬似コードのみ。図は装飾程度

### 2-2. 1 枚 1 メッセージ

- 1スライドに詰め込む情報は **1つの主張** だけ
- 「図が語る」→「1文で補足」→「次へ」のテンポを徹底
- 箇条書きは **最大4行** まで

### 2-3. カラーパレット

| 用途 | 色 | コード |
|---|---|---|
| Divide（分割） | 白背景・グレー枠 | `#ffffff` / `#d0d7de` |
| Conquer（征服） | 薄青背景 | `#e3f2fd` |
| Merge（統合） | 薄緑背景・緑枠 | `#e8f5e9` / `#4caf50` |
| Θ（シータ） | 紫 | `#9c27b0` |
| O（ビッグオー） | 青 | `#1976d2` |
| Ω（オメガ） | オレンジ | `#f57c00` |
| o（リトルオー） | 薄紫 | `#ce93d8` |
| ω（リトルオメガ） | 薄オレンジ | `#ffb74d` |
| アクセント/Q&A | 赤 | `#e53935` |

### 2-4. 図のフォーマット統一

- 形式: Mermaid（`.mmd`）を基本。Marp / reveal.js でレンダリング
- フォント: 図内テキストは日本語不可避の場合のみ日本語、基本は英数
- 枠線: 角丸（`rx:10, ry:10`）を踏襲

## 4. 第3章 Growth of Functions — 優先的に図化するスライド候補（15 個）

### 優先度 ★★★（最優先 — 話の核）

| # | スライド | 図の種類 | 内容 | 素材 |
|---|---|---|---|---|
| G-1 | なぜ漸近的記法が必要か | **Before/After グラフ** | 左: 「3n²+17n+5 と 7n²+2n+11 の正確な値」→ グラフがごちゃごちゃ。右: 「Θ(n²)」→ 1本の太いトレンドラインに。ノイズが消える様を視覚化 | 新規 |
| G-2 | Θ 記法の定義 | **挟み込みグラフ** | f(n) を c₁・g(n) と c₂・g(n) で上下から挟み込む図。n₀ より左は灰色（無視）、n₀ より右は色付き（保証区間） | 新規 |
| G-3 | O 記法の定義 | **上壁グラフ** | f(n) の上に c・g(n) の壁が覆いかぶさる図。f(n) が壁に触れない様子を明示。n₀ に垂直線 | 新規 |
| G-4 | Ω 記法の定義 | **下壁グラフ** | f(n) の下に c・g(n) の床がある図。f(n) が床より下に落ちない様子。Θ/O/Ω の3図を横並べで比較 | 新規 |
| G-5 | Θ, O, Ω, o, ω の関係まとめ | **ベン図** | 中央に Θ(g) の円、その外側に O(g) と Ω(g) の楕円が重なる。o(g) は O(g) の中で Θ(g) の外、ω(g) は Ω(g) の中で Θ(g) の外 | 新規 |

### 優先度 ★★（重要 — 理解を深める）

| # | スライド | 図の種類 | 内容 | 素材 |
|---|---|---|---|---|
| G-6 | 実数のアナロジー | **対応表カード** | 実数の「a=b, a≤b, a≥b, a<b, a>b」を左列、漸近記法を右列に並べた大きなカード5枚。1カード1対応 | 新規 |
| G-7 | Θ は "tight bound" | **挟み込みの概念図** | 上から押さえる手（O）と下から支える手（Ω）が同時にある → Θ。手のアイコンで直感的に | 新規 |
| G-8 | little-o / little-ω の「for any c」 | **段階的 c グラフ（3枚）** | c=100, c=0.001, c=10⁻¹⁰ と c を小さくしても最終的に n² < c・n³ になることを3枚のグラフで段階表示 | 新規 |
| G-9 | 比較不可能な例 | **振動グラフ** | g(n) = n^(1+sin(n)) が f(n) = n を何度も交差する図。交差部分を赤丸で強調。「n₀ をどこにおく？」の矢印付き | 新規 |
| G-10 | よくある比較の階層 | **増加度ヒエラルキー図** | O(1) → O(log n) → O(√n) → O(n) → O(n log n) → O(n²) → O(n³) → O(2ⁿ) → O(n!) を階段状に右肩上がりで描画。各段に関数名とざっくり値（n=10のとき）を添える | 新規 |

## 8. 発表のテンポを良くするための運用メモ

### 8-1. 図だけのページ（無言スライド）

**目的**: 口頭説明と図を完全に同期させる。文字がないことで聞き手が図に集中する。

| 使いどころ | 該当スライド | 想定滞在時間 |
|---|---|---|
| 分割フェーズの各ステップ | M-4（3枚） | 各3秒 |
| 統合フェーズの各ステップ | M-6（3枚） | 各5秒 |
| Θ/O/Ω のグラフ定義 | G-2, G-3, G-4 | 各10秒 |
| 増加度ヒエラルキーの全体提示 | G-10 | 5秒 |

**ルール**:
- 図のみのスライドには **タイトル以外に文字を入れない**
- タイトルは最小限（例：「分割ステップ 1/3」）
- 発表者の台本には図の説明を書くが、スライドには出さない

### 8-2. 1問1答（Q&A カード）

**目的**: 聞き手が自分で考えた直後に答えを提示することで記憶に残す。

| 使いどころ | 該当スライド | 構成 |
|---|---|---|
| 基底条件の確認 | M-7 | 質問カード → 1拍 → 答えカード |
| 具体例の判定 | G-11 | 関数ペア表示 → 5択 → 正解表示 |
| O記法の正しい使い方 | G-14 | ✅/❌ を左右に並べて即答形式 |
| Θ/O/Ω の判定練習 | G-11 の拡張 | 「n² は O(n³)?」→ 正解「はい」 |

**ルール**:
- 質問と答えは **別スライド**（クリック/スワイプで分離）
- 答えスライドには「なぜ？」を1行だけ添える
- 全体で **5〜8回** の Q&A カードを挟む（多すぎるとダレる）

### 8-3. 段階表示（プログレッシブディスクロージャー）

**目的**: 複雑な図を一度に見せず、段階的に情報を追加して理解を導く。

| 使いどころ | 該当スライド | 段階の数 |
|---|---|---|
| 分割フェーズ | M-4（divide-step1→2→3） | 3段階 |
| 統合フェーズ | M-6（merge-step1→2→3） | 3段階 |
| 再帰木 + コスト | M-5 | 4段階（ルート→レベル1→...→葉） |
| little-o の c | G-8 | 3段階（c=100, c=0.001, c=10⁻¹⁰） |
| Θ/O/Ω の定義 | G-2, G-3, G-4 | 2段階（グラフのみ→n₀ 線と範囲追加） |

**ルール**:

### 8-5. 図の表示タイミング戦略

```
[スライド構成パターン]

パターンA: 図先行（直感的理解 → 定義）
  図だけのスライド → 図＋1文 → 定義のスライド
  例: 再帰木の図 → 「各レベルのコストは常に n」→ 漸化式

パターンB: 定義先行（定義 → 図で確認）
  定義のスライド → 図による具体例 → 練習問題カード
  例: Θの定義 → 挟み込みグラフ → 「3n²+2n+1 は Θ(n²)?」

パターンC: 比較パターン（2つの図を横並べ）
  左: Before / 右: After
  例: 「3n²+17n+5 と 7n²+2n+11 の正確なグラフ」vs 「Θ(n²)」
```

**使い分け**:
- 第2章は **パターンA**（図先行）を多め。アルゴリズムは直感的に見せる
- 第3章は **パターンB**（定義先行）を多め。記法は定義を先に示してから具体例
- 導入とまとめでは **パターンC**（比較）を活用
